# Client Marketplace Flow

## Goal

Allow a client to either search for workers directly or publish a job and receive offers.

---

# Screen 1 — Client Home

Primary actions:
- Publish a job
- Search workers

Secondary content:
- Active job posts
- New offers
- Nearby suggested workers

---

# Screen 2 — Publish Job

Fields:
- Title
- Category
- Description
- Location
- Desired time
- Optional photos

UX rules:
- Keep the form short
- One clear CTA
- Encourage simple natural language
- Use location from client profile as default

CTA:
- Publish and receive offers

---

# Screen 3 — Search Workers

Features:
- Search input
- Category filters
- Worker cards
- Distance
- Rating
- Starting price
- Verification badge

Worker card should show:
- name
- category / trade
- distance
- rating
- starting price
- verification state
