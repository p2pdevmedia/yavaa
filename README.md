# Yavaa

Yavaa is a service hub for finding trusted local workers, professionals, and service providers.

The project starts in San Martin de los Andes, Argentina, with the long-term goal of expanding across Argentina, Latin America, and eventually globally.

## Mission

Yavaa exists to solve a real social problem:

People often need reliable help, but they do not know who to call, who to trust, or who is actually available.

The first goal is not profit.

The first goal is:

> Help people find trustworthy help, fast.

If the platform later creates sustainable revenue, that is welcome. But the product should always prioritize trust, usefulness, and community impact.

## Product idea

Yavaa will allow users to:

- search providers
- book services
- request emergencies
- chat internally
- manage schedules
- rate workers
- upload payment proofs
- manage commission debt

Examples of supported services:

- plumbers
- electricians
- gas technicians
- deliveries
- cleaning
- gardening
- maintenance
- psychologists via video calls
- teachers and trainers
- consultants
- local professionals

## Core principles

1. Help before profit.
2. Trust is the product.
3. Keep the experience simple.
4. Build local trust before scaling.
5. Use one app with clearly separated user modes.

## User roles

### Client
Can search, book, chat, rate, and request emergency services.

### Contractor / Provider
Can configure services, schedules, emergency availability, pricing, and manage jobs.

### Admin
Web-only operational role for moderation, disputes, debt management, and validation.

### Support
Optional role for customer support and operational follow-up.

## Platform model

The client pays the contractor directly.

Yavaa generates a platform commission as debt.

Users later pay that debt externally and upload a payment proof.

Admins validate the proof manually.

Users have a maximum allowed debt.

If the debt limit is exceeded, they can be blocked from creating or accepting new jobs.

## MVP goals

The first version should focus on:

- Next.js web app
- mobile-first responsive UI
- client mode
- contractor mode
- admin panel
- bookings
- emergency requests
- realtime chat
- ratings
- commission/debt system
- payment proof validation

Native apps:

- SwiftUI for iPhone
- Kotlin / Jetpack Compose for Android

should come after the first web experience is validated.

## Philosophy

Yavaa should prioritize:

- trust over growth
- simplicity over complexity
- usefulness over monetization
- quality over scale
- community over aggressive expansion
