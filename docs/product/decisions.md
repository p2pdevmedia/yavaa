# Product Decisions

## Launch market

Yavaa starts in Argentina, Neuquen, San Martin de los Andes.

San Martin de los Andes is the first preselected market.

## Categories

Yavaa supports many service types.

Initial groups:

- construction
- home services
- psychologists
- teachers
- massage therapists
- wellness
- delivery
- errands
- technology

Users can propose new categories.

New categories require admin approval before becoming public.

## Roles

Users register themselves.

A user can have multiple roles:

- client
- contractor
- admin
- support

Users can switch active mode inside the app.

## Contractor onboarding

Required contractor data:

- profile photo
- DNI number
- DNI front photo
- DNI back photo
- address
- first name
- last name
- phone
- service categories
- work zone

Contractors can manage their own profile, services, prices, and availability.

Public visibility and bookings require approval when needed.

## Request types

### Emergency now

- Immediate help.
- Expires after 3 hours.
- Compatible contractors are notified.
- Client chooses manually.
- If nobody answers, notifications can be retried.

### Today

- Same-day help.
- Expires after 12 hours.
- Compatible contractors are notified.
- Client chooses manually.
- If nobody answers, notifications can be retried.

### Scheduled

- Future date and time.
- Contractor has up to 24 hours to accept.
- Client chooses manually when options exist.

## Matching

Matching is assisted, not fully automatic.

The system notifies compatible contractors.

The client remains in control.

Matching should consider:

- category
- location
- availability
- urgency type
- contractor status
- debt block status
- rating later
- response time later

## Pricing

Contractors define prices.

Supported pricing types:

- per hour
- per job
- to be quoted

Emergency pricing can be configured separately.

## Principle

Yavaa helps first and monetizes later.
