# Development Phases

Yavaa is built in independent phases that later connect together.

Each phase should have:

- goal
- scope
- deliverables
- tests
- exit criteria

Recommended order:

1. foundation
2. identity and profiles
3. categories and services
4. availability
5. bookings
6. matching
7. realtime
8. chat
9. payments and debt
10. ratings and reputation
11. admin
12. agent layer
13. native apps
14. scaling

Important rule:

Do not start a phase before its required foundations are stable.

The first priority is always:

- correct data
- correct permissions
- deterministic tests
- reliable business rules
